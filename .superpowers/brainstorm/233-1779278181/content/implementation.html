实现方案

如何在不破坏 Gmeek 自动构建的前提下注入自定义设计

核心问题

Gmeek 每次构建会重新生成所有 HTML 文件（index.html、tag.html、post/*.html），直接修改这些文件会在下次 Issue 触发构建时被覆盖。需要一种机制让自定义样式在每次构建后自动重新注入。

方案：独立资源文件 + Workflow 注入

仓库结构：
docs/
├── index.html          ← Gmeek 生成
├── tag.html            ← Gmeek 生成
├── post/               ← Gmeek 生成
│   ├── *.html
├── custom.css       ← 我们创建，不会被覆盖
└── custom.js        ← 我们创建，不会被覆盖

构建流程（修改 Gmeek.yml）：
1. Gmeek.py 生成 HTML（照常）
2. 新增步骤：Python 脚本在每个 HTML 的 <head> 中
   插入 <link href="custom.css"> 和 <script src="custom.js">
3. 提交推送（照常）

结果：每次自动构建后，自定义样式自动注入 ✅

custom.css 内容规划（约 200 行）

CSS 变量 & 主题色 --sky-blue, --glass-bg, --glass-border 等
背景 & 玻璃效果 body 背景图 + backdrop-filter 玻璃卡片
Header / Footer 玻璃质感导航栏，天蓝色强调
文章列表 & 内容 玻璃卡片样式，SideNav 重写
音乐播放器 底部固定条的布局和样式
响应式 @media ≤600px / 600-900px / ≥900px 三断点

custom.js 内容规划（约 60 行）

音乐播放器控制 播放/暂停切换，进度条更新，Audio API
动态创建播放条 DOM JS 注入底部播放条 HTML（不修改 Gmeek 模板）
图片/音乐链接切换 本地 localhost 调试 → GitHub raw 链接

需要修改的文件
  • 新增 docs/custom.css
  • 新增 docs/custom.js
  • 修改 .github/workflows/Gmeek.yml
  • 不修改任何 Gmeek 生成的 HTML
  • 不修改 config.json / blogBase.json
本地调试方式
  • 直接用浏览器打开 docs/index.html
  • custom.css/js 使用相对路径自动加载
  • 图片/音频用本地文件名
  • 调试完成后替换为 GitHub 链接
